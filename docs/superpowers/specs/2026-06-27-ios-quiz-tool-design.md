# iOS 刷题工具设计规格

## 目标

做一个类似考试酷的个人刷题工具，最终交付可通过云构建生成的 IPA。用户在 iOS 16.0 设备上安装后，可以导入常见文档，解析成题库，校对后生成试卷，并进行练习、模拟考试和错题复习。

## 平台路线

- 应用形态：Web 前端 + iOS WKWebView 壳。
- 最低系统：iOS 16.0。
- 打包方式：云构建服务生成 IPA，优先使用 Codemagic，GitHub Actions 作为备选。
- 安装方式：用户下载 IPA 后，用 TrollStore 安装。
- 运行策略：前端资源内置到 IPA，核心页面离线可打开。

## MVP 范围

第一版包含以下功能：

- 题库首页：查看试卷、今日进度、错题入口。
- 导入文档：支持 PDF、docx、TXT/Markdown、Excel/CSV；老 doc 和扫描件走后端或后续增强。
- 解析流程：抽取文本/表格，本地规则拆题，疑似异常题调用 AI 修正。
- 题目校对：导入后逐题编辑题干、题型、选项、答案、解析和标签。
- 试卷生成：从导入题目发布为练习卷或模拟考试。
- 刷题：支持单选、多选、判断、填空、简答。
- 错题本：记录错误题、收藏题、待复习题。
- 设置：AI/API 接入、解析策略、题库备份、隐私与缓存、诊断日志。

不在第一版做：

- 多用户账号体系。
- 排行榜和公开题库市场。
- 复杂协作阅卷。
- 完整 OCR 本地识别。

## 文档导入

导入入口命名为“导入文档”，不要只写 Word。

支持格式：

- PDF：第一版支持可抽取文本的 PDF。扫描版 PDF 标记为需要 OCR 或后端解析。
- Word docx：解析段落、题号、选项、答案、解析。
- TXT/Markdown：适合复制出来的题库文本。
- Excel/CSV：按列导入题干、题型、选项、答案、解析。
- 老 Word doc：手机端直接解析不稳定，建议上传后端转换为 docx 或纯文本。
- 图片/扫描件：后续增强，走 OCR 后进入同一校对流程。

统一流程：

1. 选择文件。
2. 抽取文本或表格。
3. 本地规则按题型标题、题号、选项、答案和解析拆题。
4. 标记疑似异常题，例如答案缺失、选项数量异常、题型不确定。
5. 用户选择是否用 AI 修正疑似题。
6. 进入人工校对。
7. 发布为试卷。

测试样本：

- `C:/Users/liu/Desktop/复习题_20260618093745.pdf`
- 样本为 18 页可抽取文本 PDF，前几页包含单项选择题，结构类似 `1.题干（） A. ... B. ... C. ... D. ...`。

## AI/API 接入

设置页提供“AI/API 接入”入口。

配置项：

- API 地址。
- API Key。
- 模型名称。
- 测试连接。
- 解析策略：本地优先、AI 优先、混合模式。
- 隐私选项：只发送疑似题片段，不上传整份资料。

推荐默认策略：

- 混合模式。
- 先本地解析，只有异常题、格式混乱片段和用户主动修正时调用 AI。
- 第一版可以把 Key 加密或混淆后保存在设备本地。
- 更安全的正式版使用后端代理，手机端只保存自有服务地址。

AI 输出必须结构化为题目 JSON，并经过本地校验后才能进入题库。

## 题型

支持题型：

- 单选题。
- 多选题。
- 判断题。
- 填空题。
- 简答题。

判分策略：

- 单选、判断：自动判分。
- 多选：完全匹配判对，后续可加部分得分。
- 填空：第一版按文本匹配，允许多个答案。
- 简答：第一版用户自评或显示参考答案，不做强制 AI 判分。

## 数据模型

核心实体：

- `DocumentImport`：导入文件记录、格式、解析状态、错误信息。
- `Question`：题干、题型、选项、答案、解析、来源、标签、校对状态。
- `Paper`：试卷名称、题目列表、模式、时长、发布状态。
- `Attempt`：一次刷题或考试记录。
- `AnswerRecord`：用户答案、是否正确、耗时。
- `ReviewItem`：错题、收藏、复习状态。
- `ApiConfig`：API 地址、模型、密钥保存状态、解析策略。

本地存储：

- Web 前端使用 IndexedDB 存题库、试卷、进度和错题。
- iOS 壳可后续扩展 SQLite 或文件备份桥接。
- 设置页提供导出/导入 JSON 备份。

## iOS 16.0 兼容

兼容策略：

- 使用 WKWebView 壳，不依赖 iOS 17+ 专属能力。
- 前端资源打包进 IPA，首屏离线可打开。
- 题库、错题、进度存在本机，断网可刷题。
- API 失败只影响 AI 修正，不影响启动和练习。
- 文档解析分块执行：PDF 按页，Word 按段落，表格按行。
- 第一版限制单文件大小，建议 20MB 内。
- 扫描 PDF、图片 OCR、复杂版式识别放到后端。
- 设置页提供清缓存和导出诊断日志。

防闪退要求：

- 不一次性把整份大文档放入内存处理。
- 长任务显示进度，允许取消。
- 捕获 JS 错误、接口错误和解析异常。
- 解析失败时保留原文片段，允许重新解析。

## 界面方向

整体采用 iOS 系统风格：

- 大标题导航。
- 分组列表。
- 系统蓝主按钮。
- 底部 Tab：题库、刷题、错题、设置。
- 设置页集中管理 API、备份、缓存和日志。
- 导入页使用文件选择卡片和格式列表。
- 刷题页以题干卡片、选项行、底部操作按钮为主。

主要页面：

- 题库首页。
- 导入文档。
- 解析结果。
- 逐题校对。
- 试卷详情。
- 刷题/模拟考试。
- 错题本。
- 设置。
- AI/API 接入。
- 诊断日志。

## 云构建

优先方案：Codemagic。

流程：

1. Windows 本地开发应用。
2. 代码推送到 GitHub。
3. Codemagic 拉取仓库。
4. 云端安装依赖。
5. 构建 Web 前端。
6. 同步到 iOS WebView 工程。
7. 使用 Xcode 云端归档。
8. 导出 IPA artifact。
9. 用户下载 IPA，用 TrollStore 安装。

备选方案：GitHub Actions macOS runner。

云构建配置需要包含：

- Node 依赖安装。
- 前端构建。
- iOS 工程同步。
- Xcode archive。
- IPA 导出。
- 构建产物上传。

## 验收标准

MVP 完成时应满足：

- iOS 风格界面可正常浏览。
- 设置页可保存 API 配置并测试连接。
- 可导入测试 PDF 并抽取出题目文本。
- 可从 PDF 样本中解析出单选题题干和 A/B/C/D 选项。
- 解析异常题能进入校对页。
- 校对后的题目可发布为试卷。
- 单选、多选、判断、填空、简答都可进入刷题流程。
- 错题能被记录并在错题本复习。
- 断网时应用能打开并刷已有题库。
- 大文件或扫描件不会导致闪退，而是提示后端解析或暂不支持。
- 云构建配置能产出 IPA artifact。

## 测试计划

基础测试：

- 首次打开应用。
- 断网打开应用。
- 设置 API 地址、Key、模型。
- API 测试连接成功和失败。
- 导入测试 PDF。
- 导入格式不标准文本。
- 手动校对题目。
- 完成一次练习。
- 完成一次模拟考试。
- 查看错题本。
- 导出题库备份。

iOS 16.0 测试：

- iOS 16.0 真机或可用模拟环境打开。
- 连续刷题 30 分钟。
- 多次导入同一文档。
- 导入接近大小限制的文档。
- API 超时和断网场景。
- 清缓存后重启应用。

## 风险和处理

- Windows 本地不能直接生成 IPA：通过 Codemagic 或 GitHub Actions 云构建解决。
- 扫描版 PDF 识别难：第一版提示需要 OCR/后端解析，后续再做。
- AI 解析不稳定：所有 AI 输出必须进入校对流程，不能直接发布。
- API Key 泄露风险：第一版可本地保存，正式版建议后端代理。
- iOS 16 WebView 内存限制：分块解析、限制文件大小、长任务可取消。
